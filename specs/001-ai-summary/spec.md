## Clarifications

### Session 2026-01-14

- Q: aiSummary 필드가 데이터 스키마에서 필수(required)인가요, 아니면 선택(optional)인가요? → A: 필수(required) - 모든 결과물에 필드가 존재해야 하며 빈 문자열("") 허용. 빈 값일 경우 섹션을 표시하지 않음.
- Q: 300자 이상 요약의 초기 표시 상태는 어떻게 되나요? → A: 접힌 상태(collapsed)로 시작하며 "더 보기" 버튼으로 펼침.
- Q: AI 요약 섹션에 스크린 리더 지원을 위한 ARIA 라벨이 필요한가요? → A: 필요 - aria-label="AI 요약" 및 role="region" 추가.
- Q: "더 보기/접기" 버튼의 키보드 네비게이션은 어떻게 처리하나요? → A: 클릭(마우스/터치)만 지원, 별도 키보드 단축키 없음.
- Q: 300자 기준은 어떤 counting 방식인가요? → A: 글자 수(character count) 기준, 공백/문장부호 포함.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 결과물 상세 페이지에서 AI 요약 보기 (Priority: P1)

방문자는 결과물 상세 페이지에서 프로젝트 제목 하단에 표시되는 AI 생성 요약을 통해 해당 결과물의 핵심 내용을 빠르게 파악할 수 있다. 요약은 결과물의 설명, 주요 특징, 기술 스택, 활용 분야 등을 포함한다.

**Why this priority**: 방문자가 긴 프로젝트 설명을 모두 읽지 않고도 핵심 내용을 빠르게 이해할 수 있게 하여, 사용자 경험을 크게 개선한다. 이것만 있어도 AI 요약의 핵심 가치를 제공할 수 있다.

**Independent Test**: 결과물 상세 페이지에 접속하여 제목 하단에 AI 요약이 표시되는지 확인하고, 요약 내용이 결과물의 핵심 정보를 포함하는지 검증한다.

**Acceptance Scenarios**:

1. **Given** 방문자가 결과물 상세 페이지에 접속할 때, **When** 페이지가 로드되면, **Then** 프로젝트 제목 하단에 AI 요약 섹션이 표시된다
2. **Given** AI 요약이 존재하는 결과물일 때, **When** 상세 페이지가 로드되면, **Then** 요약 텍스트가 표시된다
3. **Given** AI 요약이 존재하지 않는 결과물일 때, **When** 상세 페이지가 로드되면, **Then** 요약 섹션이 표시되지 않거나 "요약이 없습니다" 메시지가 표시된다
4. **Given** 방문자가 요약을 읽을 때, **When** 요약 내용을 확인하면, **Then** 프로젝트의 목적, 주요 기능, 기술 스택 등 핵심 정보를 파악할 수 있다

---

### User Story 2 - 관리자가 AI 요약 생성 및 관리 (Priority: P1)

관리자는 각 결과물에 대해 AI 생성 요약을 추가, 수정, 삭제할 수 있다. 요약은 데이터 파일에 직접 입력하거나 관리 인터페이스를 통해 관리한다.

**Why this priority**: AI 요약 기능이 작동하려면 요약 데이터가 존재해야 한다. 관리자가 요약을 관리할 수 있어야 사용자에게 요약을 제공할 수 있다.

**Independent Test**: 결과물 데이터 파일에 요약 필드를 추가하고, 상세 페이지에서 해당 요약이 올바르게 표시되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 관리자가 새로운 결과물을 추가할 때, **When** 요약 필드에 텍스트를 입력하면, **Then** 상세 페이지에 해당 요약이 표시된다
2. **Given** 관리자가 기존 결과물의 요약을 수정할 때, **When** 요약 텍스트를 변경하면, **Then** 상세 페이지에 변경된 요약이 표시된다
3. **Given** 관리자가 요약을 삭제할 때, **When** 요약 필드를 비우면, **Then** 상세 페이지에서 요약 섹션이 숨겨진다

---

### User Story 3 - AI 요약 섹션의 시각적 구분 (Priority: P2)

AI 요약 섹션은 일반 설명 텍스트와 시각적으로 구분되어 사용자가 쉽게 인식할 수 있다. AI 아이콘이나 "AI 요약" 라벨이 표시된다.

**Why this priority**: 시각적 구분이 없으면 사용자가 요약과 원본 설명을 혼동할 수 있지만, 기능적 필수 사항은 아니다.

**Independent Test**: 상세 페이지에서 AI 요약 섹션이 다른 콘텐츠와 시각적으로 구분되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 방문자가 상세 페이지를 볼 때, **When** AI 요약 섹션을 확인하면, **Then** "AI 요약" 라벨 또는 AI 아이콘이 표시된다
2. **Given** 방문자가 상세 페이지를 볼 때, **When** AI 요약 섹션을 확인하면, **Then** 배경색이나 테두리로 구분되어 있다
3. **Given** 방문자가 모바일로 볼 때, **When** AI 요약 섹션을 확인하면, **Then** 내용이 가독성 있게 표시된다

---

### Edge Cases

- AI 요약 텍스트가 300자 이상인 경우 기본적으로 접힌 상태(collapsed)로 시작하며 "더 보기" 버튼으로 펼쳐지고, 다시 "접기"로 접을 수 있다
- AI 요약 텍스트가 비어있거나 null인 경우 요약 섹션을 표시하지 않는다
- AI 요약에 마크다운 형식이 포함된 경우 올바르게 렌더링한다
- AI 요약에 특수문자나 이모지가 포함된 경우 올바르게 표시된다
- 요약 로딩에 실패한 경우 에러 메시지를 표시하지 않고 섹션을 숨긴다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 결과물 상세 페이지의 프로젝트 제목 하단에 AI 요약 섹션을 표시해야 한다
- **FR-002**: 시스템은 AI 요약 필드가 빈 문자열("")이 아닌 경우에만 요약 섹션을 표시해야 한다
- **FR-003**: 시스템은 AI 요약 섹션에 "AI 요약" 라벨 또는 AI 아이콘을 포함해야 하며, 스크린 리더 지원을 위해 aria-label="AI 요약"과 role="region"을 제공해야 한다
- **FR-004**: 시스템은 결과물 데이터 스키마에 요약 필드(aiSummary)를 필수(required) 필드로 포함해야 하며, 빈 문자열("")을 허용해야 한다
- **FR-005**: 시스템은 AI 요약 텍스트에서 마크다운 형식을 지원해야 한다
- **FR-006**: 시스템은 AI 요약 텍스트가 300자(글자 수, 공백/문장부호 포함) 이상인 경우 기본적으로 접힌 상태(collapsed)로 표시하고 클릭(마우스/터치)으로 동작하는 "더 보기/접기" 버튼을 제공해야 한다
- **FR-007**: 시스템은 AI 요약 섹션을 다른 콘텐츠와 시각적으로 구분해야 한다 (배경색, 테두리, 아이콘 등)
- **FR-008**: 시스템은 AI 요약 섹션을 반응형으로 제공하여 모바일에서도 정상 작동해야 한다

### Key Entities

- **AI 요약 (AI Summary)**: 결과물의 핵심 내용을 요약한 텍스트. 필수 필드(required)로 빈 문자열("")을 허용한다. 속성으로는 요약 텍스트, 생성일, 생성자(AI/수동)를 포함한다
- **결과물 (Project)**: 기존 결과물 엔티티에 aiSummary 필드가 필수 항목으로 추가됨 (빈 값 허용)

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 방문자는 상세 페이지 로드 후 1초 이내에 AI 요약을 확인할 수 있다
- **SC-002**: 80%의 방문자가 AI 요약을 통해 프로젝트 핵심 내용을 30초 이내에 파악할 수 있다
- **SC-003**: AI 요약이 없는 결과물의 경우 요약 섹션이 표시되지 않아야 한다 (100%)
- **SC-004**: 모바일에서도 AI 요약이 가독성 있게 표시되어야 한다

## Assumptions

- AI 요약은 WCAG 2.1 AA 수준의 접근성을 준수해야 한다 (ARIA 라벨, 적절한 시맨틱 마크업)
- AI 요약은 관리자가 수동으로 생성하거나 외부 AI 도구를 사용하여 생성한다
- AI 요약은 한국어로 작성된다
- AI 요약 텍스트 길이는 일반적으로 100~500자 사이다
- 요약 생성은 실시간 API가 아니라 빌드 시점에 데이터에서 로드된다

## Out of Scope

- 실시간 AI 요약 생성 API 연동
- 요약 텍스트의 자 번역 기능
- 사용자가 AI 요약을 편집할 수 있는 기능
- 요약 품질에 대한 피드백 시스템
- 여러 요약 버전 관리
